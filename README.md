# 流放之路2 物价追踪器

Windows 本地版 POE2 查价与价格记录工具，主要面向国服玩家。它以本地数据库为核心，结合 poe2db 经济数据、玩家主动导入的实时价格、截图 OCR 和快捷键浮窗，帮助玩家更快判断通货、符文、预兆、技能宝石等物品的参考价格。

## 主要功能

### 物价列表

物价列表用于查看、搜索、排序、收藏和对比本地价格数据。

支持：

- 按物品名搜索，支持空格分隔多个关键词
- 按名称、价格、单位、走势、来源、更新时间等列排序
- 切换显示单位
- 查看趋势百分比和简化走势图
- 收藏、置顶和隐藏/显示表格列
- 按来源和分页查看数据

趋势规则：

- 有有效实时价格时，价格和趋势优先使用实时价格数据。
- 完全没有有效实时价格时，才回退到 poe2db 或普通本地历史记录。
- 趋势优先比较较近历史数据，避免长期使用最早记录造成误导。
- 历史点不足时，只显示可解释的百分比或“暂无”，不强行绘制曲线。

### 收藏列表

收藏列表用于集中查看重点关注物品。它和物价列表保持一致的搜索、排序、趋势、单位切换和显示列逻辑。

### F4 快捷查价

默认快捷键：`F4`

在游戏中悬停物品并复制物品信息后，按 `F4` 可以在鼠标附近显示查价浮窗。浮窗会展示价格、单位、趋势、来源、时间和点赞信息，并根据屏幕边界自动调整位置，避免跑出屏幕。

### Ctrl+Space 聚焦搜索

默认快捷键：`Ctrl+Space`

快速呼出搜索框，输入关键词即可查看匹配价格。搜索框支持快速切换显示单位，适合不打开主窗口时临时查价。

### F1 截图查价

默认快捷键：`F1`

框选截图后，软件会通过 OCR 识别截图中可查价的物品，并显示查价结果浮窗。这个功能面向“尽可能从截图里找出可查价物品”，适合符文、赛季特殊玩法物品、截图文字等场景。

F1 的边界：

- 主要识别可查价物品名。
- 会结合本地标准库进行匹配，减少 UI 文案和噪声误识别。
- 不承担实时价格导入的交易比例解析职责。
- 结果浮窗会保留价格、单位、趋势、来源、时间和点赞。

### F6 实时价格导入

默认快捷键：`F6`

用于框选游戏内市场或交易报价区域，把看到的真实报价整理为一条实时价格记录。识别完成后可以确认提交，并可同步到共享数据服务。

当前特性：

- 支持物品名称占两行的情况，例如技能宝石和部分特殊物品。
- 一般情况下只识别一条价格行。
- 允许修正物品名称和交易方向。
- 不允许手动改写识别到的价格比例。
- 物品名称修改会做合理性校验，避免完全乱编。
- 如果已有可信实时价格，可直接点赞已有记录。

### 同步经济数据

可从 poe2db 经济页同步价格数据：

https://poe2db.tw/cn/Economy

同步结果写入本地数据库，作为没有实时价格时的重要参考来源。

### 实时价格同步

玩家可以主动同步自己导入的实时价格，也可以拉取其他玩家共享的实时价格。软件会尽量增量、分批处理同步数据，降低单次同步失败和重复写入的风险。

实时价格有点赞机制。查询时，如果某物品存在满足点赞阈值的实时价格记录，会优先使用这些记录；如果完全没有有效实时价格，才回退到 poe2db 或普通本地记录。

### 手动记录

可以手动输入物品名称、价格、单位和来源，写入本地数据库。适合补充个人观察到的价格，或记录暂时没有同步到的数据。

### 配置与本地维护

配置页支持：

- 快捷键设置
- 窗口最小化/关闭行为
- 主题切换
- 全局字体大小
- 价格小数位数
- OCR 后端和 CPU 线程
- 截图缓存数量
- 是否显示截图识别详情
- 实时价格最低点赞阈值
- 打开缓存位置
- 清理本地数据
- 检查更新

启动时会对配置和数据库做基础校验。如果配置文件或数据库损坏，软件会提示用户选择是否重新初始化，并说明可能导致本地数据或偏好丢失。

## 本地数据

软件数据默认保存在：

```text
%LOCALAPPDATA%\PoE2PriceTracker
```

常见文件：

- `prices.sqlite3`：本地价格数据库
- `config.json`：用户配置
- `screenshots\`：截图缓存

可以在配置页打开缓存位置或清理本地数据。清理本地数据不会删除软件本体。
